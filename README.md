# Color Palette Generator

A Coolors-style color palette generator built with Next.js 16, React 19, and Tailwind CSS v4. Generates harmonious color palettes using perceptual color science (OKLCH + CIEDE2000), with a fully interactive UI for exploring, editing, and saving palettes.

Fully generated by Claude Code.

## Features

### Palette generation
- Press **Space** to generate a new palette using one of four color harmony rules: analogous, complementary, split-complementary, or triadic
- All generation happens in **OKLCH color space** via [culori](https://culorijs.org), ensuring perceptually even hue offsets and vibrant, non-muddy colors
- Every generated palette enforces a minimum **ΔE2000 distance of 20** between all color pairs, and a minimum lightness standard deviation of 0.12 across the palette
- The initial palette is **generated server-side** and passed as props, so the first render is meaningful rather than an empty shell

### Swatches
- **2–10 swatches**, each filling an equal share of the viewport height
- **Lock** any swatch to exclude it from regeneration — locked swatches display a filled badge icon so the state is unambiguous at a glance
- **Remove** a swatch (minimum 2 enforced)
- **Insert** a new swatch between any two adjacent ones — the inserted color is interpolated in OKLCH space from its two neighbours
- **Click the hex code** to copy it to clipboard
- **Color picker** button opens a card with a native color wheel and a hex text input, both kept in sync — pick visually or type a hex value

### Shades
- Click the **shades icon** on any swatch to overlay 18 horizontal colour slabs, ranging from OKLCH lightness 0.95 (near-white) to 0.10 (near-black) in 5% steps
- Click any slab to replace the swatch with that shade

### Saving palettes
- **Save** the current palette with a custom name — stored in `localStorage`
- Saved palettes appear in a **dropdown** in the navbar with colour chip previews; click any to load it
- **Download** the current palette as a `palette.json` file
- **Import** a previously exported JSON file to restore a palette

## Tech stack

| Concern | Choice |
|---|---|
| Framework | Next.js 16 (App Router) |
| UI | React 19 |
| Styling | Tailwind CSS v4 |
| Color math | [culori](https://culorijs.org) v4 — OKLCH generation, CIEDE2000 distance, HSL conversion |
| State | React hooks only — no external state library |
| Persistence | `localStorage` (saved palettes) + JSON file export/import |

## Color generation details

`lib/colors.ts` works entirely in OKLCH until the final hex conversion step:

1. A seed hue is chosen at random (or inherited from the first locked swatch)
2. A harmony rule is picked at random — offsets are applied to the OKLCH hue, which is perceptually uniform so 120° really does look like a third of the colour wheel
3. Each candidate color is validated: chroma ≥ 0.06, lightness in [0.15, 0.90]
4. Candidates are accepted only if their CIEDE2000 distance from all already-accepted colors is ≥ 20 (threshold relaxes after repeated failed attempts)
5. If the palette's lightness standard deviation is below 0.12, lightness values are redistributed to evenly-spaced targets while hue and chroma are preserved
6. Colors are converted to hex only as the final step
